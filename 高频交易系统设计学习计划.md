# 低时延 / 高频交易系统设计学习计划

> 更新时间：2026-03-17 16:53:18 CST  
> 当前起点：已完成约前 3 周 C++ 基础，默认从第 4 周以后继续推进  
> 学习目标：最终能独立做出一个简化版低时延交易系统，但必须按“小项目驱动、逐步升级”的方式前进

## 学习总原则

1. 不一开始做完整交易系统。
2. 优先做小项目，逐步升级。
3. 每个阶段先做出“能跑、能测、能解释”的成果。
4. 每周都要有代码产出，不只看仓库。
5. 每个阶段都要重视 benchmark、延迟、工程化和数据结构选择。
6. 学习顺序固定为：
   - 工程化 C++ 小项目
   - 数据结构与内存 / 性能意识
   - 并发、线程池、 ring buffer、消息传递
   - 订单簿与撮合
   - 协议层、性能分析、最小交易链路
7. 不过早进入 DPDK、RDMA、FIX 深水区，除非前面基础已经完成。

## 第 4 周到第 12 周按周学习路径

### 第 4 周：工程化 C++ 小项目补强

**学习目标**

- 巩固 RAII、move、错误处理、目录结构、CMake、测试、README 写作。
- 建立“代码不只是能跑，还要能维护、能解释、能扩展”的意识。

**推荐看的仓库 / 文件**

- `modern-cpp-tutorial`
- `cppbestpractices`

**本周小项目**

- `mini_logger`
  - 支持同步日志
  - 支持日志级别
  - 支持文件输出
  - 支持简单滚动切分
  - 支持基础配置

**建议接口**

```cpp
enum class LogLevel { Debug, Info, Warn, Error };

class Logger {
public:
    void log(LogLevel level, std::string_view message);
};
```

**本周验收标准**

- 能用 `CMake` 构建
- 有 `tests/`
- 有清晰的 `include/`、`src/`、`bench/`、`tests/` 目录
- README 能解释设计取舍
- 至少写 1 个小 benchmark，比如测每秒日志写入量

### 第 5 周：数据结构与 benchmark 入门

**学习目标**

- 建立“性能来自数据结构和内存布局”的直觉。
- 理解顺序访问、随机访问、插入删除、哈希查找的成本差异。

**推荐看的仓库 / 文件**

- `Algorithms-And-Data-Structures`
- `cppbestpractices`

**本周小项目**

- `benchmark_harness`
  - 对比 `vector`、`deque`、`list`
  - 对比 `map`、`unordered_map`
  - 对比不同字符串解析方式

**本周验收标准**

- 能输出平均耗时、吞吐
- 能比较至少 3 种容器
- 能写出结论，不只给数字
- 能说明“为什么这个结构更快 / 更慢”

### 第 6 周：RingBuffer 与内存意识

**学习目标**

- 掌握固定容量队列、head / tail、缓存友好、对齐与容量设计。
- 初步建立低分配、低抖动的数据通路意识。

**推荐看的仓库 / 文件**

- `QuantumLeaps/lock-free-ring-buffer`
- `kjeffery/lock_free_ring_buffer`

**本周小项目**

- `ring_buffer`
  - 先做单线程版本
  - 再设计 SPSC 版本接口

**建议接口**

```cpp
template <typename T, std::size_t Capacity>
class RingBuffer {
public:
    bool try_push(const T& value);
    bool try_pop(T& out);
    std::size_t size() const;
    constexpr std::size_t capacity() const;
};
```

**本周验收标准**

- 写功能测试和边界测试
- benchmark 对比 `std::deque`
- 能解释固定容量结构为什么更适合低时延场景
- 能说明 power-of-two 容量设计的意义

### 第 7 周：ObjectPool 与文件回放器

**学习目标**

- 理解对象复用、避免频繁分配、批量回放、顺序 I/O 的价值。

**推荐看的仓库 / 文件**

- `Algorithms-And-Data-Structures`
- 复用你前几周写的 benchmark 框架

**本周小项目**

- `object_pool`
- `replay_tool`
  - 从 CSV 回放订单事件流

**建议接口**

```cpp
template <typename T>
class ObjectPool {
public:
    T* acquire();
    void release(T* obj);
};
```

**本周验收标准**

- pool 支持 `acquire/release`
- 回放器能顺序读事件并调用处理函数
- benchmark 对比对象池与 `new/delete`
- README 说明对象池适用场景和风险

### 第 8 周：线程池与任务模型

**学习目标**

- 掌握任务提交、worker loop、停止机制、异常传播、future 基础。
- 理解线程池适合解决什么问题，不适合解决什么问题。

**推荐看的仓库 / 文件**

- `bshoshany/thread-pool`

**本周小项目**

- `simple_thread_pool`

**建议接口**

```cpp
class ThreadPool {
public:
    explicit ThreadPool(std::size_t thread_count);
    ~ThreadPool();

    template <typename F, typename... Args>
    auto submit(F&& f, Args&&... args);
};
```

**本周验收标准**

- 支持 `submit()`
- 有 CPU 任务测试和 I/O 模拟任务测试
- benchmark 输出任务吞吐和线程数变化曲线
- 能解释线程池不适合直接作为超低时延主链路核心的原因

### 第 9 周：SPSC Queue 与跨线程消息传递

**学习目标**

- 掌握单生产者单消费者模型、原子变量、基本 memory order 语义。
- 理解消息传递比共享复杂对象更适合作为核心链路。

**推荐看的仓库 / 文件**

- `QuantumLeaps/lock-free-ring-buffer`
- `kjeffery/lock_free_ring_buffer`

**本周小项目**

- `spsc_queue`
- `message_bus_demo`

**建议消息结构**

```cpp
enum class MessageType { MarketData, Order, Cancel, Trade };

struct Message {
    MessageType type;
    std::uint64_t id;
    std::uint64_t ts_ns;
    std::int64_t price;
    std::int64_t qty;
};
```

**本周验收标准**

- 一个线程生产，一个线程消费
- 消息结构固定大小
- 输出吞吐、平均延迟、p50、p99
- 能解释为什么 SPSC 在低时延链路里很常见

### 第 10 周：价格档位聚合器

**学习目标**

- 理解 price level、bid / ask、数量聚合、快照和增量更新。

**推荐看的仓库 / 文件**

- `SLMolenaar/orderbook-simulator-cpp`
- `Johannestampere/order_book_simulator`

**本周小项目**

- `price_level_aggregator`

**建议接口**

```cpp
enum class Side { Buy, Sell };

class PriceLevelAggregator {
public:
    void on_add(std::int64_t price, std::int64_t qty, Side side);
    void on_cancel(std::int64_t price, std::int64_t qty, Side side);
    auto top_n(Side side, std::size_t n) const;
};
```

**本周验收标准**

- 能维护 bid / ask 档位
- 能打印 Top-of-Book 和 Top-5
- 用回放器喂入数据
- benchmark 每秒事件处理量

### 第 11 周：最小订单簿

**学习目标**

- 掌握 ADD / CANCEL / MODIFY、order id 索引、price-time priority。
- 理解 price level 与 order node 的关系。

**推荐看的仓库 / 文件**

- `SLMolenaar/orderbook-simulator-cpp`
- `Johannestampere/order_book_simulator`
- `PushkarRJain/Low-Latency-Order-Book-Matching-Engine`

**本周小项目**

- `mini_order_book`

**建议接口**

```cpp
struct AddOrder {
    std::uint64_t order_id;
    Side side;
    std::int64_t price;
    std::int64_t qty;
};

class OrderBook {
public:
    void add(const AddOrder& order);
    void cancel(std::uint64_t order_id);
    void modify(std::uint64_t order_id, std::int64_t new_qty);
};
```

**本周验收标准**

- 单元测试覆盖典型订单生命周期
- 支持回放一批订单流
- 输出订单簿状态摘要
- 能说明为什么选择某种数据结构组合

### 第 12 周：最小撮合器

**学习目标**

- 理解成交规则、price-time priority、partial fill、trade event。

**推荐看的仓库 / 文件**

- `SLMolenaar/orderbook-simulator-cpp`
- `Johannestampere/order_book_simulator`
- `PushkarRJain/Low-Latency-Order-Book-Matching-Engine`

**本周小项目**

- `matching_engine_minimal`

**建议事件输出**

```cpp
struct Trade {
    std::uint64_t resting_order_id;
    std::uint64_t incoming_order_id;
    std::int64_t price;
    std::int64_t qty;
};
```

**本周验收标准**

- 至少支持一种清晰的限价单撮合路径
- 能产生 `TRADE` 事件
- 对吞吐与延迟做 benchmark
- 能画出事件流：`input -> book -> match -> trade report`
